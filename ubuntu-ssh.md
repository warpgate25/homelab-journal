Развёртывание Ubuntu Server + настройка SSH-доступа

## Цель
Настроить домашнюю лабораторию для практики системного администрирования, 
разобраться с базовыми командами Linux и удалённым доступом к серверу.

## Окружение
- Хост: Windows 11, Lenovo Legion 5
- Гипервизор: Oracle VirtualBox
- Гостевая ОС: Ubuntu Server 24.04 LTS

## Что сделано
- Установлен VirtualBox, настроена виртуальная машина (2 CPU, 2 ГБ RAM, 25 ГБ диск)
- Развёрнута Ubuntu Server 24.04 LTS
- Изучены базовые команды диагностики системы:
whoami
uname -a
df -h
free -h

- Изучена работа с системным журналом:
journalctl -n 20
journalctl -p err
journalctl -u ssh

- Установлен и настроен OpenSSH-сервер:
sudo apt install openssh-server -y
sudo systemctl start ssh
sudo systemctl enable ssh

- Настроен сетевой режим VirtualBox на Bridged Adapter (сервер получил IP 
  в локальной сети, доступен как отдельное устройство)
- Проверено удалённое подключение по SSH с двух разных устройств:
  - Windows (PowerShell)
  - macOS (Terminal)

## Проблемы и решения

**Проблема:** Ubuntu 26.04 LTS вызывала kernel panic при установке в VirtualBox 
(баг совместимости версии VirtualBox с новым ядром).
**Решение:** Перешёл на предыдущую стабильную LTS-версию — Ubuntu 24.04, 
установка прошла без ошибок.

**Проблема:** SSH-подключение с хоста выдавало "Connection timed out".
**Решение:** VirtualBox по умолчанию использует режим сети NAT, который 
изолирует виртуалку от внешних подключений. Сменил на Bridged Adapter — 
сервер стал доступен по IP в локальной сети.

**Проблема:** После смены сети SSH выдавал "Permission denied".
**Решение:** Оказалась опечатка в логине пользователя при подключении.

## Итог
Поднят рабочий Linux-сервер с удалённым доступом по SSH, проверена 
диагностика через логи и системные утилиты. Следующий шаг — установка 
и настройка Docker.
